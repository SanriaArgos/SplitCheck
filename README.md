# Проект: SplitCheck

Прототип приложения для учёта чеков и разделения позиций между участниками.

## 1. Описание проекта

В рамках экзамена необходимо реализовать упрощённое веб-приложение на Django, которое позволит:
1. Зарегистрировать нового пользователя и авторизоваться.
2. Создать чек, загрузив фотографию (изображение) и указав названия позиций с ценами на одной странице.
3. Указать имена участников чека (в виде обычных текстовых полей).
4. Просматривать список своих чеков (страница «Мои чеки»).

### Основная идея
- Пользователь заходит в приложение, регистрируется/авторизуется.
- Переходит на страницу «Мои чеки».
- Создаёт новый чек:
  - Загрузить фотографию чека (необязательно обрабатывать содержимое автоматически).
  - Ввести несколько позиций (название, цена).
  - Указать участников (просто список имён, без привязки к реальным аккаунтам).
- После сохранения чек появляется в списке «Мои чеки», и при нажатии на него видны все данные (фото, позиции, участники).

**Примечание**: Распределять, кто сколько должен, не обязательно. Главное — показать базовое создание чека, привязку к пользователю, формы, загрузку изображений и отображение результата.

## 2. Требования к сервису

1. **Регистрация и авторизация**  
   - Использовать стандартную систему Django Authentication (логин, логаут).
   - После входа пользователь должен попадать на страницу «Мои чеки».

2. **Страница «Мои чеки»**  
   - Показывать список чеков, созданных текущим пользователем.
   - Каждая запись содержит краткую информацию (название, дата создания).
   - Кнопка «Создать чек».

3. **Создание чека**  
   - При нажатии на «Создать чек» переходим на форму:
     1. Загрузка файла (фото чека).
     2. Название чека (например, «Поход в кафе»).
     3. Список позиций (название, цена). Можно реализовать добавление позиций динамически (по желанию) или ограничиться фиксированным количеством строк.
     4. Участники чека: несколько полей (например, три-четыре), куда можно вписать имена. Можно просто текстовые поля (e.g. «Андрей», «Ваня», «Маша»).
   - Данные сохраняются и привязываются к текущему пользователю.
   - После сохранения — редирект на страницу «Мои чеки» или детальную страницу чека.

4. **Просмотр деталей чека**  
   - Отдельная страница, где видны:
     - Заголовок чека.
     - Загруженная фотография.
     - Список позиций.
     - Имена участников.
   - Необязательно делать возможность редактирования, но это может быть плюсом.

5. **Загрузка изображений**  
   - Настроить `MEDIA_ROOT` и `MEDIA_URL` в settings.py, чтобы хранить загруженные файлы.
   - Убедиться, что при создании чека изображение сохраняется и отображается на детальной странице.

6. **Структура проекта и вёрстка**  
   - Можно использовать чистый HTML или подключить Bootstrap для более аккуратного вида.
   - Наличие главного меню (navbar) с ссылками на «Мои чеки» и «Выход» (если пользователь авторизован).

## 4. Инструкция по установке и запуску

1. **Убедитесь, что у вас установлен Python 3.10**  
   Проверить версию Python можно командой:
   ```bash
   python3 --version
   ```
   или
   ```bash
   python --version
   ```
   (в зависимости от настроек в системе).

2. **Склонировать репозиторий**:
   ```bash
   git clone git@github.com:<YourName>/Repo.git
   ```
   Перейдите в каталог репозитория:
   ```bash
   cd <RepositoryName>
   ```

3. **Создать виртуальное окружение** (например, `venv`):
   ```bash
   python3 -m venv venv
   ```
   или для Python 3.10:
   ```bash
   python3.10 -m venv venv
   ```

4. **Активировать виртуальное окружение**:
   - Linux/Mac:
     ```bash
     source venv/bin/activate
     ```
   - Windows:
     ```bat
     venv\Scripts\activate
     ```

5. **Установить зависимости** из файла `requirements.txt`:
   ```bash
   pip install -r requirements.txt
   ```

6. **Выполнить миграции** (создать/обновить базу данных):
   ```bash
   python manage.py migrate
   ```

7. **Запустить сервер** разработки:
   ```bash
   python manage.py runserver
   ```
   По умолчанию приложение будет доступно по адресу:
   ```
   http://127.0.0.1:8000/
   ```

## 6. Работа с Git и создание ветки `check`

1. **Создать новую ветку** `check` для внесения изменений:
   ```bash
   git checkout -b check
   ```
2. **Внести нужные изменения** (редактировать файлы, добавлять функционал и т. д.).
3. **Добавить изменённые файлы в индекс**:
   ```bash
   git add .
   ```
4. **Закоммитить изменения**:
   ```bash
   git commit -m "Добавлена функциональность для работы с чеками"
   ```
5. **Отправить (push) ветку на GitHub**:
   ```bash
   git push origin check
   ```

6. **Создать Pull Request**  
   - Зайдите на страницу репозитория в GitHub.  
   - Найдите кнопку «Compare & pull request» (или «New pull request») для ветки `check`.  
   - Оформите Pull Request (укажите цель слияния в основную ветку, например, `main` или `master`).
